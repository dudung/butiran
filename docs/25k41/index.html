  • Updating a GitHub repository or simply a repo, which has submodules, still confuses me. Especially when to update the parent repo and when to update the subomdule. It is a little bit clearer after a discussion, where update submodule repo first then update the parent repo to point to the new submodule revision 1. The last update is not necessary after every first update, but after several first updates only to assure that the parent repo uses the new submodule revision.
